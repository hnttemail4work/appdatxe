# Booking Platform - Full Stack Setup Guide\n\n## Overview\n\nThis is a complete long-distance car booking platform built with:\n- **Backend**: Laravel 11 with Sanctum authentication\n- **Frontend**: React 19 with TypeScript and Tailwind CSS\n- **API Communication**: Axios with custom client wrapper\n\n## Prerequisites\n\n- PHP 8.2+\n- Node.js 18+\n- MySQL 8.0+ (or any compatible database)\n- Composer\n- npm or yarn\n\n## Backend Setup\n\n### 1. Install Dependencies\n\n```bash\ncd backend\ncomposer install\n```\n\n### 2. Configure Environment\n\n```bash\ncp .env.example .env\nphp artisan key:generate\n```\n\nEdit `.env` with your database credentials:\n\n```\nDB_CONNECTION=mysql\nDB_HOST=127.0.0.1\nDB_PORT=3306\nDB_DATABASE=booking_platform\nDB_USERNAME=root\nDB_PASSWORD=your_password\n```\n\n### 3. Create Database\n\n```bash\n# Create empty database\nmysql -u root -p -e \"CREATE DATABASE booking_platform;\"\n```\n\n### 4. Run Migrations\n\n```bash\nphp artisan migrate\n```\n\nThis creates 11 tables:\n- `users` - Customers, operators, and admins\n- `trip_routes` - Available trip routes\n- `vehicles` - Fleet vehicles\n- `schedules` - Trip schedules\n- `bookings` - Customer bookings\n- `seat_reservations` - Seat holds and reservations\n- `payment_transactions` - Payment records\n- `merchant_profiles` - Operator KYC verification\n- `platform_settings` - Global configuration\n- `booking_audits` - Audit trail\n- `payouts` - Operator payouts\n\n### 5. Seed Sample Data\n\n```bash\nphp artisan db:seed\n```\n\nCreates:\n- 3 trip routes (HCM→VungTau, HCM→DaLat, HCM→MuiNe)\n- 9 users (3 customers, 3 operators, 3 admins)\n- 4 vehicles (2 per operator)\n- 8 schedules\n- Sample merchant profiles and settings\n\n### 6. Start Development Server\n\n```bash\nphp artisan serve\n```\n\nAPI will be available at `http://localhost:8000/api`\n\n## Frontend Setup\n\n### 1. Install Dependencies\n\n```bash\nnpm install\n# or\nyarn install\n```\n\n### 2. Configure Environment\n\nCreate `.env` file (copy from `.env.example`):\n\n```bash\ncp .env.example .env\n```\n\nEnsure it has:\n```\nVITE_API_BASE_URL=http://localhost:8000/api\n```\n\n### 3. Start Development Server\n\n```bash\nnpm run dev\n```\n\nFrontend will be available at `http://localhost:3000`\n\n## API Endpoints Reference\n\n### Authentication\n\n```\nPOST   /api/auth/register          - Register new user\nPOST   /api/auth/login             - Login\nGET    /api/auth/me                - Get current user (requires token)\nPOST   /api/auth/logout            - Logout (requires token)\n```\n\n### Customer Routes (role: customer)\n\n```\nGET    /api/customer/trips/search        - Search trips with filters\nGET    /api/customer/trips/{id}          - Get trip details with seat map\nGET    /api/customer/bookings            - List customer bookings\nPOST   /api/customer/bookings            - Create booking\nGET    /api/customer/bookings/{id}       - Get booking details\nGET    /api/customer/bookings/{id}/payment-payload  - Get payment options\nPOST   /api/customer/bookings/{id}/confirm-payment  - Confirm payment\nDELETE /api/customer/bookings/{id}       - Cancel booking\n```\n\n### Operator Routes (role: operator)\n\n```\nGET    /api/operator/vehicles            - List vehicles\nPOST   /api/operator/vehicles            - Create vehicle\nGET    /api/operator/vehicles/{id}       - Get vehicle\nPUT    /api/operator/vehicles/{id}       - Update vehicle\nDELETE /api/operator/vehicles/{id}       - Delete vehicle\n\nGET    /api/operator/schedules           - List schedules\nPOST   /api/operator/schedules           - Create schedule\nGET    /api/operator/schedules/{id}      - Get schedule\nGET    /api/operator/schedules/{id}/seat-grid  - Get seat availability\n\nGET    /api/operator/passengers          - List passengers for upcoming trips\n```\n\n### Admin Routes (role: admin)\n\n```\nGET    /api/admin/merchants              - List merchants\nPOST   /api/admin/merchants/{id}/approve - Approve merchant\nPOST   /api/admin/merchants/{id}/reject  - Reject merchant\n\nGET    /api/admin/commission-settings    - Get commission rate\nPUT    /api/admin/commission-settings    - Update commission rate\n\nGET    /api/admin/orders                 - List all orders with audit\nGET    /api/admin/orders/{id}            - Get order details\n\nGET    /api/admin/analytics/revenue      - Get revenue analytics\n```\n\n## Frontend Project Structure\n\n```\nsrc/\n├── components/              # React components\n├── context/ token storage in localStorage\n- Token refresh on app load\n- Auto-logout on 401 Unauthorized\n\n### Customer Features\n- Search trips with filters (city, date, time, vehicle type)\n- View available seats in real-time\n- Create bookings with seat selection\n- See booking history (upcoming and past)\n- Process payments with mock providers (VietQR, MoMo)\n- Cancel bookings with automatic seat release\n\n### Operator Features\n- Manage fleet vehicles (add, edit, delete)\n- Create and manage trip schedules\n- View real-time seat availability\n- See list of passengers for upcoming trips\n- Accept or reject customer bookings\n\n### Admin Features\n- Approve/reject operator KYC verification\n- Configure global commission percentage\n- View all bookings with audit trail\n- Generate revenue analytics reports\n- See per-operator breakdown of revenue and payouts\n\n## Business Logic\n\n### Pricing\n- Base price per route\n- Booking total = base_price × number_of_seats\n- Deposit = total_price × 30%\n\n### Commission\n- Platform takes configured percentage (default 10%) of gross revenue\n- Operator payout = gross_revenue - platform_commission\n\n### Seat Holds\n- Seats automatically held for 15 minutes after booking creation\n- Hold expires if payment not confirmed within 15 minutes\n- Expired holds automatically release seats\n\n### Payment Flow\n- Customer creates booking with selected seats\n- System generates mock payment payload (VietQR/MoMo)\n- Customer confirms payment with transaction reference\n- Payment confirmed → booking status changes to \"confirmed\"\n- Seat holds converted to permanent bookings\n\n## Testing Accounts\n\nDefault seeded users (password: 'password'):\n\n**Customers:**\n- customer1@example.com\n- etc.\n2. **Frontend changes**: Edit files in `src/`\n3. **API changes**: Update types in `src/types/api.ts` and API client in `src/services/api.ts`\n4. **Database schema**: Update migrations in `backend/database/migrations/`, then run `php artisan migrate`\n\n## Deployment\n\n### Backend\n\n```bash\ncd backend\ncomposer install --no-dev\nphp artisan migrate --force\nphp artisan cache:clear\nphp artisan config:cache\n```\n\n### Frontend\n\n```bash\nnpm run build\n# dist/ folder contains production build\n```\n\n## Troubleshooting\n\n### CORS Issues\nMake sure backend is
